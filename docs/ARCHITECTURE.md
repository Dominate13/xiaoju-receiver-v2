## 核心模块

### 1. PushReceiver (推送接收)

负责接收小桔充电平台的订单推送请求。

**职责**：
- 接收 HTTP POST 请求
- 解析加密数据
- 调用 PushHandler 处理

### 2. PushHandler (推送处理)

核心业务逻辑处理模块。

**职责**：
- AES-128-CBC 数据解密
- HMAC-SHA256 签名验签
- 订单数据解析
- 去重检查（基于 start_charge_seq）
- 数据库持久化

### 3. SyncWorker (同步工作器)

定时同步任务执行器。

**职责**：
- 定时查询待同步记录
- 幂等性检查（本地 + 氦云）
- 字段映射转换
- 调用氦云 CreateBizObject API
- 失败重试机制

### 4. H3yunClient (氦云客户端)

氦云 OpenAPI 封装。

**职责**：
- HTTP 请求封装
- 认证头处理
- 业务对象创建
- 重复查询

## 数据流

### 推送接收流程

```
1. 小桔平台 POST /api/push
2. PushReceiver 路由接收
3. PushHandler 处理：
   a. 检查签名 (HMAC-SHA256)
   b. 解密数据 (AES-128-CBC)
   c. 解析 JSON
   d. 检查重复
   e. 保存数据库
4. 返回处理结果
```

### 同步流程

```
1. 定时触发（默认30秒）
2. 查询 pending_sync 记录
3. 逐条处理：
   a. 检查氦云是否存在
   b. 字段映射转换
   c. 调用氦云 API
   d. 更新同步状态
4. 失败记录标记并重试
```

## 数据库设计

### push_records 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| start_charge_seq | VARCHAR(64) | 订单号（唯一索引） |
| connector_id | VARCHAR(32) | 充电枪ID |
| station_id | VARCHAR(32) | 充电站ID |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| total_power | FLOAT | 总电量 |
| total_money | FLOAT | 总金额 |
| sync_status | VARCHAR(20) | 同步状态 |
| sync_at | DATETIME | 同步时间 |
| sync_error | TEXT | 同步错误 |
| raw_data | TEXT | 原始数据 |
| sig_valid | INTEGER | 验签状态 |

### sync_status 状态值

- `pending_sync`：待同步
- `synced`：已同步
- `failed`：同步失败

## 安全设计

### 加密机制

- **算法**：AES-128-CBC
- **填充**：PKCS7
- **密钥管理**：环境变量配置

### 验签机制

- **算法**：HMAC-SHA256
- **比较**：timing-safe 对比
- **密钥管理**：环境变量配置

## 扩展性

### 多平台支持

可通过扩展 PushHandler 支持其他充电平台：

1. 定义平台适配器接口
2. 实现小桔充电适配器
3. 注册到工厂类

### 数据库迁移

当前使用 SQLite，生产环境建议迁移到 PostgreSQL：

1. 修改 DB_URL
2. 更新 SQLAlchemy 配置
3. 执行 schema 迁移
